# x-post — 記事からX投稿文を生成する

公開済みの記事をもとに、X（Twitter）投稿用の文章を3案生成する。

## やること

1. **対象記事を特定する**
   - 引数がある場合 → スラッグや記事タイトルで `site/src/content/posts/` から該当ファイルを探す
   - 引数がない場合 → `site/src/content/posts/` 内の最新ファイル（ファイル名日付が最も新しいもの）を使う

2. **記事を読み込む**
   - frontmatter（title, description, category, tags, date）を取得
   - 本文の内容も把握する（冒頭・主要な気づき・結論）

3. **X投稿文を3案生成する**
   - 各案は280文字以内
   - 記事URL `https://slothfuldevlog.com/posts/<slug>` を含める
   - カテゴリ別ハッシュタグを2〜3個つける
   - ブランドボイスに従う

4. **3案を出力し、どれを投稿するか確認する**
   - 3案を表示した後、「案1・案2・案3のどれを投稿しますか？（スキップする場合は「なし」）」と聞く

5. **選ばれた案をXに投稿する**
   - まず `scripts/` に移動して依存パッケージをインストール（初回のみ）:
     ```
     cd scripts && npm install && cd ..
     ```
   - 選ばれた案のテキスト（URL・ハッシュタグ含む）を1つの文字列にして投稿:
     ```
     node scripts/post-to-x.js "[投稿テキスト全文]"
     ```
   - 投稿成功後、ツイートURLを表示する

## ブランドボイスのルール

- 「力まない」「本音で」「一緒に考える」トーン
- 宣伝臭を出さない。気づき・発見・問い から始める
- 専門用語を使いすぎない（非エンジニアにも伝わる言葉で）
- 一人称は「私」または省略
- 硬すぎず、崩しすぎず

## 投稿案のバリエーション

| 案 | 切り口 | 例 |
|---|---|---|
| 案1 | シンプル告知型 | 「新記事を書きました。〇〇の話。」 |
| 案2 | 問いかけ型 | 「〇〇ってできると思ってなかったんだけど、やってみたらできた。」 |
| 案3 | 発見・気づき共有型 | 「〇〇をやってみてわかったこと：〜〜。」 |

## カテゴリ別ハッシュタグ

- `devlog`: `#Vibecoding` `#ClaudeCode` `#個人開発`
- `body-mind`: `#パーソナルトレーナー` `#トレーニング` `#機能解剖学`
- `culture`: `#読書` `#美術` `#印象派`
- `essays`: `#エッセイ` `#気づき`
- `projects`: `#個人開発` `#プロジェクト`

## 出力フォーマット

```
━━━ 案1（シンプル告知型） ━━━
[投稿文]

[URL]

[ハッシュタグ]

━━━ 案2（問いかけ型） ━━━
[投稿文]

[URL]

[ハッシュタグ]

━━━ 案3（発見・気づき共有型） ━━━
[投稿文]

[URL]

[ハッシュタグ]
```

